<paramSpec>

<paramSpec> specification of a parameter of a model behaviour
Moduletagdocs — Documentation Elements
Attributesatt.global (@xml:id, @n, @xml:lang, @xml:base, @xml:space) (att.global.rendition (@rend, @style, @rendition)) (att.global.linking (@corresp, @synch, @sameAs, @copyOf, @next, @prev, @exclude, @select)) (att.global.analytic (@ana)) (att.global.facs (@facs)) (att.global.change (@change)) (att.global.responsibility (@cert, @resp)) att.identified (@ident, @predeclare, @module) (att.combinable (@mode) (att.deprecated (@validUntil)) )
Contained by
tagdocs: paramList
May contain
core: desc gloss
tagdocs: altIdent equiv
Note

The paramSpec element is used to provide a specification of a parameter to be used in conjunction with a model element's behaviour attribute. In the example provided a behaviour attribute of link has two parameters: content and link. Using a desc attribute within a paramSpec is optional but recommended practice.

Example
<valItem ident="link">
 <desc>create hyperlink</desc>
 <equiv name="a"/>
 <paramList>
  <paramSpec ident="content"/>
  <paramSpec ident="link">
   <desc>link url</desc>
  </paramSpec>
 </paramList>
</valItem>
